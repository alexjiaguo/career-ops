# Career-Ops for Codex / Antigravity

Read `CLAUDE.md` for all project instructions, routing, and behavioral rules. They apply equally to Codex and Antigravity.

Key points:
- Reuse the existing modes, scripts, templates, and tracker flow — do not create parallel logic.
- Store user-specific customization in `config/profile.yml`, `modes/_profile.md`, or `article-digest.md` — never in `modes/_shared.md`.
- Never submit an application on the user's behalf.
- **Obsidian-native**: All JD files, evaluations, and tracking live in the Obsidian vault. Read `modes/obsidian-bridge.md` for paths.
- **Bases tracker**: The tracker at `00_Strategy/Tracker_2026.base` reads JD file frontmatter automatically. No separate applications.md needed.
- **Skill delegation**: Resume generation → `tailored-resume-generator` skill. Interview prep → `interview-prep` + `mock-interview-transcript` skills.

For Codex-specific setup, see `docs/CODEX.md`.
